# Technical Context

Architecture, design decisions, and known limitations of **SQL Schema Visualizer**. Read this before contributing.

## Overview

A 100% client-side single-page app. SQL DDL is parsed in the browser into an in-memory schema model, which is laid out with dagre and rendered as an interactive React Flow diagram. There is no backend, no network call, and no persistence beyond the browser's `localStorage`.

## Parser design (`src/lib/parser.ts`)

The parser is **hand-rolled**, not a SQL grammar library. Rationale:

- Multi-dialect tolerance beats strict correctness for a *visualization* tool. A real grammar for one dialect chokes on vendor clauses (`ENGINE=InnoDB`, `AUTOINCREMENT`, backticks); juggling four grammars is worse.
- Goal is a diagram, not validation: **skip-on-unknown, never throw**. A partially-understood dump still yields a useful picture, with `warnings[]` surfaced in the UI.

Pipeline:

1. **`stripComments`** ‚Äî removes `--`, `#`, `/* */` while respecting string literals (used for the code-parsing pass only; comments are recovered separately).
2. **`splitStatements`** ‚Äî splits on top-level `;`, tracking quote and parenthesis depth.
3. **`CREATE TABLE`** ‚Äî extracts the table name (quotes/brackets/schema-prefix stripped via `cleanIdent`), splits the body on top-level commas, classifies each part as a column definition or a table constraint.
4. **Column parse** ‚Äî name, type (leading token incl. parenthesised length, `UNSIGNED`, array `[]`), `NOT NULL`, `PRIMARY KEY`, `UNIQUE`, `DEFAULT`, inline `REFERENCES`.
5. **Constraints** ‚Äî table-level `PRIMARY KEY (...)`, `UNIQUE (...)`, `FOREIGN KEY (...) REFERENCES ...`; plus `ALTER TABLE ... ADD FOREIGN KEY`.
6. **`associateComments`** ‚Äî a second physical-line walk that re-attaches `--` / `#` comments to the right table/column.

### Comment association heuristic

`associateComments` walks lines, tracking the current table and last seen column:

- A comment on a `CREATE TABLE` header line ‚Üí table comment.
- A comment on a column-definition line ‚Üí that column.
- A **comment-only** line (continuation) ‚Üí appended to the last seen column, else the table.

Consequence: a wrapped comment block whose continuation lands on the line *after* the header binds to the first column, not the table. The text is never lost, but its anchor can shift by one row. This is an accepted tradeoff for a forgiving line-based pass ‚Äî a full positional parser would be far heavier for marginal gain.

## Rendering & interaction (`Canvas.tsx`)

- `baseNodes` / `baseEdges` are memoised from `schema`; `layout()` runs dagre and returns positioned nodes. The layout effect re-fits the view on schema/direction change.
- **Search** dims non-matching nodes (match = table name *or* any column name) and animates connected FK edges.
- **Focus** (click-to-navigate): a store `focus` object with an incrementing `nonce` retriggers an effect that `setCenter`s on the node and flags the focused column. `setCenter` runs inside `requestAnimationFrame` *after* the node update.
- **Gotcha (documented):** the React Flow `fitView` *prop* re-fits the viewport on every node change and silently cancels `setCenter`. It is intentionally **not** set as a prop ‚Äî initial fit is done manually via the layout effect's rAF `fitView()` call. Do not re-add the `fitView` prop without rethinking focus navigation.
- **Hover popovers** escape the node bounds; `.react-flow__node:hover { z-index: 1000 }` (in `index.css`) lifts the hovered node above siblings so popovers aren't occluded.
- **Navigation model:** `panOnScroll` + `panOnScrollMode=Free` + `zoomOnScroll=false` + `zoomActivationKeyCode=['Meta','Control']` = Figma/trackpad feel. Magic Mouse has no pinch hardware; zoom there is ‚åò/Ctrl+scroll, double-click, or the controls.

## State (`store.ts`)

Single Zustand store. Persisted to `localStorage`: `dbviz.sql` (last SQL), `dbviz.comments` (`off` | `inline` | `hover`, with legacy `0`/`1` migration). `search`, `focus`, `direction` are session-only.

## Known limitations

- Not a real SQL grammar ‚Äî exotic DDL (generated columns, deeply nested parens, CTE-in-DDL, partition clauses) parses loosely.
- Comment continuation anchoring can shift by one row (see heuristic above).
- Hover popover anchors right (`left-full`); near the right viewport edge it may extend off-screen ‚Äî pan to read.
- No live database introspection (by design ‚Äî client-only, privacy-first).
- Layout is dagre-only; very wide schemas produce long horizontal spans (toggle vertical or pan).
